# データベース設計

## usersテーブル

| カラム名     | 説明                     |
| ------------ | ------------------------ |
| name         | ユーザー名               |
| email        | ログイン用メールアドレス |
| password     | パスワード               |
| image        | プロフィール画像         |
| introduction | 自己紹介                 |

---

## blogsテーブル

| カラム名    | 説明           |
| ----------- | -------------- |
| title       | タイトル       |
| body        | 本文           |
| image       | サムネイル画像 |
| category_id | カテゴリID     |

---

## categoriesテーブル

| カラム名 | 説明       |
| -------- | ---------- |
| name     | カテゴリ名 |

---

## catsテーブル

| カラム名      | 説明                   |
| ------------- | ---------------------- |
| name          | 猫の名前               |
| breed         | 種類                   |
| gender        | 性別（1:オス, 2:メス） |
| date_of_birth | 生年月日               |
| image         | 画像                   |
| introduction  | 紹介文                 |

---

## blog_catテーブル（中間テーブル）

| カラム名 | 説明     |
| -------- | -------- |
| blog_id  | ブログID |
| cat_id   | 猫ID     |

---

## 設計意図

- ブログと猫を多対多で関連付けることで、柔軟なコンテンツ構成を実現
- カテゴリを分離することでブログの分類を容易にした
- ユーザーにプロフィール情報を持たせ、拡張性を確保
